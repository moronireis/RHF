Vamos começar.

Proposta aprovada. Agora é o momento de alinhar expectativas, definir acessos e garantir que as primeiras 3 semanas entreguem o Quick Win: currículos automáticos.

Objetivo

01 — Recap

O que foi aprovado

Recapitulando o escopo para garantir que estamos todos alinhados.

Gerador de Currículo com IA

De 15 minutos para 2. Currículo montado automaticamente com dados do Pandapé + conversa do ChatGuru.

Sync Pandapé ↔ ChatGuru

Candidato aprovado na triagem? Contato criado automaticamente no ChatGuru. Funis sincronizados em tempo real.

Abertura Inteligente + Dashboard

Formulário recebido → IA gera descritivo + roteiro. Dashboard com métricas de performance.

Atendimento Assistido por IA

Respostas sugeridas no ChatGuru baseadas no contexto da vaga e perfil do candidato. Recrutador valida e envia.

02 — Acessos críticos

O que precisamos agora

Sem esses acessos, não conseguimos iniciar. São o bloqueio número 1.

!

Acesso à API do Pandapé

Precisamos das credenciais de API (token ou chave) para acessar candidatos, vagas e movimentações de funil. Solicitar ao suporte do Pandapé ou no painel administrativo.

Bloqueante — sem isso não começamos
!

Acesso à API do ChatGuru

Token de API + configuração de webhooks. Precisamos ler conversas, criar contatos e enviar arquivos (PDF do currículo). Geralmente fica em Configurações → Integrações.

Bloqueante — sem isso não começamos
!

Usuário de teste no Pandapé

Um usuário com permissão de leitura + escrita para que possamos testar as integrações sem afetar dados reais. Ideal: ambiente de staging ou usuário com escopo limitado.

Bloqueante
03 — Templates e dados

Materiais para o gerador de CV

Precisamos do modelo visual atual para reproduzir fielmente no gerador automático.

!

Template atual do currículo (Word ou PDF)

O modelo que a equipe usa hoje para formatar os currículos. Precisamos da versão editável (Word) com as fontes, cores e layout da marca RHF. Se tiver mais de uma versão, enviar todas.

Bloqueante — define o layout do PDF gerado
!

3 a 5 exemplos de currículos já finalizados

Currículos que foram enviados a clientes recentemente. Serve de referência para o tom do resumo profissional, nível de detalhe e formatação esperada. Podem anonimizar se preferirem.

Importante — calibra a qualidade da IA
!

3 exemplos de descritivos de vaga

Descritivos que a equipe já produziu (seja via ChatGPT ou manual). Precisamos entender o padrão de linguagem, estrutura e nível de detalhe que o Rodrigo espera.

Importante — calibra o gerador de descritivos
!

Prompts atuais do ChatGPT

Rodrigo mencionou que já usa prompts no ChatGPT para gerar resumos e descritivos. Se puder compartilhar esses prompts, conseguimos manter a mesma qualidade e adicionar a automação por cima.

Desejável — acelera o desenvolvimento
04 — Processos

Entender o fluxo real

Precisamos de 30-60 minutos com quem opera o dia a dia para mapear cada passo.

!

Sessão de mapeamento com recrutador(a)

Uma chamada de 30-60 min com quem faz o processo de currículo hoje. Queremos ver na tela: como abre o Pandapé, como copia dados, como cola no ChatGPT, como monta no Word, como envia pelo ChatGuru. Ver é diferente de ouvir.

Importante — semana 1
!

Mapeamento dos estágios do funil

Lista dos estágios usados no Pandapé (ex: Triagem, Qualificado, Entrevista, Apresentado, Contratado) e no ChatGuru (ex: Novo Contato, Em Atendimento, CV Enviado). Precisamos saber o "de-para" entre os dois.

Importante — define a lógica de sync
!

Acesso ao formulário MS Forms de abertura de vaga

O formulário que o cliente preenche para solicitar uma vaga. Precisamos ver os campos, formato e como os dados chegam hoje (email, Excel, OneNote?).

Desejável — fase 3

07 — Comunicação

Como vamos nos comunicar

Canal único, ritmo claro, sem ruído.

Grupo WhatsApp

Canal principal para dúvidas rápidas, envio de materiais e updates diários. Rodrigo + equipe RHF + Moroni + Tiago.

Call semanal (30 min)

Toda semana, mesmo horário. Status do projeto, próximos passos, bloqueios. Rápido e objetivo.

Documento de acompanhamento

Compartilhado com o Rodrigo. Atualizado após cada call com o que foi feito, o que vem a seguir e o que está pendente.

Regra de ouro

Se algo travar, avisamos em 24 horas, não na call semanal. Quanto antes soubermos de um bloqueio, antes resolvemos.
