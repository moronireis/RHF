Análise Interna: Projeto RHF Talentos

Planejamento de custos, precificação, riscos e oportunidades para o piloto de automação com IA na RHF Talentos Vale dos Sinos.

01Resumo executivo da call

Cliente: Rodrigo Silva — franqueado RHF Talentos, unidade Vale dos Sinos (9 anos de operação).

Rede: 300+ unidades, ~90% usam Pandapé (ATS oficial). CRM do franqueador foi cancelado.

Modelo de negócio: 100% por sucesso. Só ganha quando coloca candidato. Ticket médio R$ 2.500. Volume: 50-60 processos/mês, 10-20 currículos/dia.

Ferramentas atuais do Rodrigo

  • Pandapé — ATS oficial da franquia (tem API). Funil de vagas e candidatos.
  • ChatGuru — WhatsApp API oficial. Contrato de 12 meses (outubro 2025 → outubro 2026). Tem API, webhooks, campos personalizados, automações.
  • ChatGPT — Conta com API, mas usa manual (copiar/colar). Gera descritivos de vaga e resumos profissionais.
  • MS Office 365 — Forms para abertura de vagas, OneNote para dados, Word templates para currículo.

Dor prioritária (Rodrigo confirmou na call)

Preparação do currículo. Pandapé → ChatGPT → Word → PDF → ChatGuru. São 15-20 min por currículo, 10-20x ao dia. É o processo que mais consome tempo e mais gera erros por copia e cola.

O que Rodrigo deixou claro

  • Não quer mudanças bruscas — está cumprindo metas e não quer desestabilizar.
  • Não quer substituir o ChatGuru agora (contrato ativo, está funcionando).
  • Quer gerar mais caixa este ano para investir mais depois.
  • Está aberto a ser case de validação e apresentar para o franqueador.
  • Outro franqueado já criou um sistema próprio (RecrutaFlow), mas Rodrigo considera arriscado investir nele.

03Sugestão de precificação

Três opções de pricing para discutir internamente. O Rodrigo é uma franquia de porte médio, não é enterprise — mas o potencial de rede justifica um preço agressivo no piloto.

Opção A — Piloto agressivo

Setup: R$ 3.500 (ou 3x R$ 1.167)

Recorrência: R$ 497/mês

Prós: Barreira de entrada baixa, facilita o sim. Margem menor no setup mas recorrência compensa em 6-8 meses. Ideal se priorizamos o case para a rede.

Opção B — Padrão

Setup: R$ 5.000 (ou 3x R$ 1.667)

Recorrência: R$ 497/mês

Prós: Margem saudável no setup. Valor justo para o escopo. É o que está na proposta atual do cliente.

Opção C — Premium

Setup: R$ 7.000 (ou 4x R$ 1.750)

Recorrência: R$ 697/mês

Prós: Margem alta. Inclui dashboard avançado + 60 dias de suporte. Risco: pode assustar num primeiro projeto.

Opção D — Revenue Share

Setup: R$ 2.000 (custo mínimo)

Recorrência: R$ 297/mês + 5% sobre incremento de faturamento

Prós: Alinha incentivos. Risco: difícil medir incremento. Só funciona com dashboard de métricas rodando.

Recomendação interna: Opção B (R$ 5.000 + R$ 497/mês)

Equilíbrio entre margem saudável e acessibilidade para o Rodrigo. Com desconto à vista (R$ 4.500), continua viável. A recorrência de R$ 497 cobre custos de IA + infra + 2h de suporte e ainda sobra margem. Se priorizarmos o case para a rede, a Opção A também é válida — margem menor no curto prazo, mas potencial de 300+ unidades compensa.

Nota sobre margem negativa no setup

Nos cenários A e B, o setup não cobre 100% do custo de desenvolvimento. Isso é intencional se o objetivo for conquistar o case para a rede RHF. O payback vem de:
(1) Recorrência mensal — break-even em 10-15 meses
(2) Replicação para rede — custo de dev próximo de zero no segundo cliente em diante
(3) Aprendizado — domínio do nicho de recrutamento que tem milhares de empresas no Brasil

Se o objetivo for margem positiva já no setup, ir com Opção C ou reduzir escopo.

05Riscos e mitigações

RiscoProbabilidadeImpactoMitigação
API do Pandapé limitada ou mal documentada Média Alto Fazer discovery técnico ANTES de fechar proposta. Pedir acesso à documentação da API ao Rodrigo. Se a API não tiver endpoints necessários, avaliar scraping ou redução de escopo.
ChatGuru não permite integração que precisamos Média-Baixa Médio ChatGuru tem API documentada e webhooks. Risco é menor. Validar se permite enviar PDF via API e ler histórico de conversa.
Escopo cresce além do previsto (scope creep) Alta Médio Rodrigo tem muitas ideias. Definir escopo fechado por fase. Quick Win = currículo automático. Só avança para próxima fase após validação. Contrato com escopo explícito.
Rodrigo não adota (resistência da equipe) Média Médio Ele já usa ChatGPT e está aberto a automação. Risco é mais na equipe de recrutadores. Incluir treinamento e 30 dias de acompanhamento.
Franqueador escolhe outra solução Média Alto (para expansão) Não dependemos do franqueador para o piloto. Se escolherem outra solução de atendimento (não ChatGuru), nossa integração precisaria adaptar. Manter arquitetura modular.
RecrutaFlow (concorrente interno) ganha tração Baixa Médio Rodrigo descartou — considerou arriscado investir em ferramenta de outro franqueado. Franqueador também não homologou. Mas monitorar.
Qualidade da IA no resumo profissional Baixa Médio Rodrigo já usa ChatGPT para isso e está satisfeito com a qualidade. Nosso motor seria mais rápido e automatizado, não precisa ser melhor que o GPT-4 — precisa ser igual e sem copia e cola.
Dados sensíveis (CPF, documentos) Média Alto Rodrigo já tem prompts que filtram dados sensíveis. Implementar filtro automático + revisão humana obrigatória antes de enviar ao cliente. LGPD compliance.

Risco crítico: Discovery técnico não feito

A proposta atual assume que as APIs do Pandapé e ChatGuru têm os endpoints necessários. Antes de fechar, precisamos de 2-4 horas de discovery técnico para validar: (1) Pandapé API permite ler candidatos e dados do funil, (2) ChatGuru API permite ler conversas e enviar arquivos, (3) Formato dos dados é compatível com o pipeline que planejamos. Se uma das APIs não atender, o escopo e o preço mudam.

06Aberturas e oportunidades

Oportunidade 1: Rede RHF (300+ unidades)

O maior ativo desse projeto não é o piloto em si — é o acesso à rede. Rodrigo é franqueado respeitado (9 anos), já validou e apresentou soluções para o franqueador antes. Se o piloto funcionar, ele leva para a rede. 300 unidades × R$ 497/mês = R$ 149.100/mês de recorrência potencial. Mesmo com 10% de adesão no primeiro ano, são R$ 14.910/mês.

Oportunidade 2: Produto para nicho de recrutamento

A dor do Rodrigo não é exclusiva da RHF. Qualquer empresa de recrutamento que use ATS + WhatsApp tem o mesmo problema. O produto desenvolvido no piloto pode virar um SaaS vertical para o nicho de R&S.

  • Mercado: milhares de consultorias de RH no Brasil
  • Pandapé é líder na América Latina — integração com eles é um diferencial
  • Poucos players conectam ATS + WhatsApp + IA de forma integrada

Oportunidade 3: Modelo de parceria com franqueador

Rodrigo mencionou que outro fornecedor (universidade corporativa) conseguiu se tornar recomendado pelo franqueador com um modelo de indicação + comissionamento. Isso é replicável:

  • Franqueador indica → franqueado assina → franqueador recebe % ou fee fixo
  • Franqueador tem interesse: melhor performance da rede = mais royalties
  • Jantar com franqueador na próxima quinta (feira de franquias em Porto Alegre) — timing é agora

Oportunidade 4: Expansão de escopo futuro

  • Triagem automática com IA: Candidato se aplica → IA pontua perfil vs requisitos → pre-triagem automática. Elimina trabalho manual de análise.
  • Chatbot de atendimento inicial: Candidato entra pelo WhatsApp → chatbot faz perguntas de qualificação → só passa para recrutador quem estiver qualificado.
  • Portal do cliente: Empregador acompanha status das vagas em tempo real (em vez de perguntar no WhatsApp).
  • Relatório automático para franqueador: Métricas agregadas da rede — um produto premium que o franqueador pagaria diretamente.

Oportunidade 5: ChatGuru como parceiro

O ChatGuru está tentando ser o sistema de atendimento recomendado da rede RHF. Se nós somos os que adicionam a camada de IA por cima do ChatGuru, isso vira um pacote: ChatGuru (atendimento) + u4digital (IA). Possibilidade de co-selling ou parceria comercial com o ChatGuru.

Veredicto geral

Projeto de alto potencial estratégico e margem apertada no curto prazo. O valor real está na porta que abre para 300+ unidades, não no piloto em si. Recomendação: fechar com preço agressivo-a-padrão (Opção A ou B), executar rápido (Quick Win em 3 semanas), documentar resultados, e usar o case para abrir a rede.

Próximo passo concreto: Discovery técnico das APIs (2-4h) → decisão de pricing → enviar proposta final → criar grupo WhatsApp → iniciar.
